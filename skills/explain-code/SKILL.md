## What I do
- Summarize what a file/function does in plain language
- Trace inputs, outputs, and external side effects (network, fs, env)
- Point out non-obvious behavior, risks, and likely bugs

## When to use me
Use this when onboarding to a codebase, reviewing unfamiliar code, or preparing
a PR description. Prefer reading the relevant files via the read tool first.
