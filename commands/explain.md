description: Explain how code works in plain language
---

Explain the code in the specified files or the current directory.

For each file or module:
1. **Purpose** — What does this code do? Why does it exist?
2. **Architecture** — How is it structured? What patterns are used?
3. **Data flow** — What goes in, what comes out?
4. **Dependencies** — What does it depend on? What depends on it?
5. **Gotchas** — Any edge cases, non-obvious behavior, or potential issues?

Use plain language. Avoid jargon unless necessary. Assume the reader is a
developer but unfamiliar with this specific codebase.
